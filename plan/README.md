# Implementation Plans

All implementation plans are saved here with consistent naming.

## Naming Convention

```
[YYYY-MM-DD]-[feature-name]-[feature-id].md

Examples:
- 2026-01-19-user-onboarding-F001.md
- 2026-01-19-google-apple-signin-F002.md
- 2026-01-19-email-password-auth-F003.md
```

## Structure

Each plan file contains:

1. **Feature Overview**
   - Feature ID, name, priority, day
   - Description and acceptance criteria

2. **Implementation Strategy**
   - Components to create
   - Database changes
   - API endpoints / Server Actions needed
   - State management plan

3. **File Structure**
   - Which files to create/modify
   - Folder organization

4. **Step-by-Step Implementation**
   - Numbered steps from requirements
   - Code patterns to follow

5. **Testing Checklist**
   - Test steps from feature_list.json
   - States to verify (loading, empty, success, error)
   - Edge cases

6. **Blockers & Dependencies**
   - What must be done first
   - External dependencies

## How to Use

1. **Before building a feature:** Read the corresponding plan file
2. **During implementation:** Follow the step-by-step guide
3. **After implementation:** Use testing checklist to verify
4. **On blockers:** Check dependencies section

## Updating Plans

Plans are created before implementation and should be updated with:
- Decisions made during implementation
- Issues discovered
- Alternative approaches tried
- Time estimates (optional, only if tracking)
